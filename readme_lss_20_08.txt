Задача №8 Метод решения через построение LU разложения

В main_lss_20_08.c находится разбор командной строки: проверяется введена команда (и какая именно команда),
путь к файлу для ввода и вывода или нет. Если нет, то выводится код ошибки. Если файл был не введён, то для ввода и вывода
используются файлы по умолчанию : lss_20_08_in.txt и lss_20_08_out.txt

Перед считыванием из файла, он проверяется на пустоту. Далее считывается n (если получается считать)  и проверяется
положительное число это или нет.Если нет, то выводится ошибка. Если n положительно, то выделяются память на массив для
матрицы A и и вектора В, далее они считываются (если получается считать).

Потом выделяются память для вектора X и tmp. Память для tmp высчитывается по формуле:
    n*n+n

Далее идёт переход на вычисление решения системы в файле lss_20_08.c
Проверяется сначала размер матрицы для проверки её на вырожденность:
    - Если матрица размера 1, то проверяется равенство её элемента на ноль, если не ноль, то сразу высчитвается решение
    - Если матрица размера 2, то идёт подсчёт определителя матрицы по формуле: a11*a22-a12*a21
    - Иначе высчитывается определитель матрицы методом Гаусса. Для этого матрица A копируется в tmp. Матрица сводится к
    треугольному виду и высчитывается произведение её диагоналей.

После подсчёта определителя, проверяется равенство на ноль. Если ноль, то программа возвращает код ошибки -1
Иначе обнуляется матрица tmp,  высчитываются матрицы L (у L на диагонали 1) и U, они хранятся как 1 матрица в tmp в виде:
    u_11 u_12 .... u_1n
    l_21 u_22 .... u_2n
    l_31 l_32 .... u_3n
    ...................
    l_n1 l_n2 .... u_nn

Далее высчитывается система LY=B прямой подстановкой. Значения вектора Y записываются в конец массива tmp.
После этого высчитывается система UX=Y обратной подстановкой.

В конце, если подпрограмма не вывела код ошибки, то решение выводится в файл.

Коды ошибок:
    -2 - Ошибка открытия файла
    -3 - Набор команды не верен
    -4 - Файл для ввода пуст
    -5 - В файле для ввода указаны неверные значения
    -6 - Введено неверное кол-во файлов в командной строке (когда добавлен только 1 путь)
